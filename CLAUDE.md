# termgrid 项目规范

## 开发规范

### TDD 严格执行

- **所有功能必须 TDD**：先写测试，再写实现，严格 RED-GREEN-REFACTOR
- **单元测试**：每个模块都要有充分的单测覆盖，包括边界情况和错误路径
- **集成测试**：模块间交互、PTY 生命周期、Git 检测等都需要集成测试
- 测试放在模块文件底部的 `#[cfg(test)] mod tests` 中，集成测试放在 `tests/` 目录

### 项目结构

追求成熟、清晰、可复用的架构：
- 模块职责单一，接口清晰
- 公共类型和 trait 定义在独立模块，便于复用
- 避免模块间循环依赖
- 代码可复用优先于快速实现

### Rust 规范

- 修改后使用 `cargo check` 验证编译，`cargo test` 验证测试
- 禁止全局 `cargo fmt` 或 `cargo clippy --fix`
- clippy 警告仅修复本次变更涉及的代码
